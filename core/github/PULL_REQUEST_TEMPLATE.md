# Pull Request

## Problem observed

What broke, failed, or behaved incorrectly? Describe the concrete symptom.

## Evidence / source

What evidence shows this is a real problem? Link issues, logs, screenshots, tests, user reports, or repository files when available.

## Why this belongs in this repo

Explain why the change belongs here instead of a downstream app, fork, private configuration, or separate integration.

## Alternatives considered

What other approaches did you consider, and why were they rejected?

## Gates run

List the verification commands, results, skipped gates, and residual risk.

## Human review confirmation

- [ ] A human reviewed the complete diff before submission.

## Single coherent change confirmation

- [ ] This PR contains one coherent change.
- [ ] This PR does not bundle unrelated refactors, cleanup, formatting churn, or opportunistic fixes.

PRs with fabricated problem statements, speculative fixes, or bundled unrelated changes will be closed without review.
